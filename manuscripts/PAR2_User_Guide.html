1. Introduction & Core Concepts

What is the PAR(2) Discovery Engine?

The PAR(2) Discovery Engine is a statistical platform for analyzing gene expression time-series data. Its central idea: fit a second-order autoregressive model — AR(2) — to each gene's expression over time, then extract the eigenvalue modulus |λ| as a single number that captures how strongly that gene's expression "persists" from one time point to the next.

The core question: Instead of asking "Is this gene rhythmic?" (yes/no), AR(2) asks "How persistent is this gene's temporal pattern?" — a continuous measurement that captures both oscillation strength and memory.

What is AR(2)?

An autoregressive model of order 2 means: the expression value at time t depends on the values at times t−1 and t−2, plus noise. Mathematically: x(t) = β₁ · x(t−1) + β₂ · x(t−2) + ε

The two coefficients β₁ and β₂ define the gene's "dynamical signature." From these, we compute the eigenvalue modulus |λ|, which ranges from 0 to 1:

|λ| ValueInterpretationBiological Example
0.0 – 0.3Weak persistence — expression decays quicklyNoise-driven, rapidly turning over genes
0.3 – 0.6Moderate persistence — some temporal memoryTarget genes responding to signals
0.6 – 0.8Strong persistence — robust temporal patternClock-controlled output genes
0.8 – 1.0Very strong persistence — self-sustaining oscillationCore clock genes (Per2, Arntl, Clock)

The Clock > Target Hierarchy

The central finding of the PAR(2) framework is the clock > target eigenvalue hierarchy: core circadian clock genes (Per2, Arntl, Clock, Cry1, etc.) consistently show higher |λ| values than their downstream target genes (Myc, Ccnd1, Wee1, etc.) across tissues, species, and conditions.

This is not just a statistical observation — it reflects biology. Clock genes form self-reinforcing transcription-translation feedback loops, giving them stronger temporal memory. Target genes respond to clock signals but lack their own sustaining loops.

2. Quick Start (5-Minute Tour)

Follow these steps to get oriented in 5 minutes:

Step 1: Open the Dashboard

The home page shows summary statistics and a list of pre-loaded datasets. You'll see 38+ datasets spanning mouse, human, baboon, and plant tissues.

Step 2: Select a Dataset & Run Analysis

Pick any dataset (start with Mouse Liver (GSE11923) — it has the most time points). Click "Run Analysis". The engine computes AR(2) eigenvalues for all clock and target genes in seconds.

Step 3: Read the Results

You'll see eigenvalue bar charts, time-series plots, and the clock-vs-target comparison. The gap between clock and target eigenvalues is the key metric — a larger gap means stronger circadian hierarchy.

Step 4: Explore Root-Space

Navigate to /root-space to see where every gene lands in coefficient space. Try the Waddington landscape and phase portrait overlays.

Step 5: Try Your Own Data

Go to /discovery-engine and upload a CSV file with time-series data. The engine will compute AR(2) eigenvalues with full diagnostics.

Tip: Use the navigation menu (top-left hamburger icon) to jump between all pages. Each page has a "How To" helper that explains what it does.

3. The Dashboard

The Dashboard (/) is the main hub. It shows pre-loaded gene expression datasets and lets you run AR(2) analysis interactively.

3.1 Selecting Datasets

The platform includes 38+ datasets from public repositories (primarily NCBI GEO). Each dataset card shows:

  • Species — Mouse, Human, Baboon, Arabidopsis
  • Tissue — Liver, SCN, Lung, Blood, Skin, Colon, etc.
  • GEO Accession — e.g., GSE11923 (clickable link to the source data)
  • Time Points — Number and spacing of measurements
  • Condition — Normal, disease, treatment, etc.

3.2 Running an Analysis

Select clock and target genes from the gene lists, then click "Run Analysis". The engine fits AR(2) models to every selected gene and returns eigenvalues within seconds.

You can customize which genes to analyze — the defaults include the core clock genes (Per2, Arntl, Clock, Cry1, Cry2, Nr1d1, Nr1d2, Per1) and representative target genes (Myc, Ccnd1, Wee1, Cdkn1a, etc.).

3.3 Reading Results

The results panel shows several views:

  • Eigenvalue Bar Chart — Each gene's |λ| as a horizontal bar. Clock genes (cyan) should cluster higher than target genes (pink/amber).
  • Time-Series Plot — Raw expression data over time for selected genes, showing the oscillatory patterns.
  • AR(2) Coefficients — The raw β₁ and β₂ values and model fit (R²).
  • Cross-Context Comparison — Run the same genes across multiple datasets to compare tissue-specific persistence.
  • Edge Case Diagnostics — Reliability checks for trend contamination, sample size, nonlinearity, and boundary proximity.
Tip: You can download results as CSV, export charts as images, and generate a printable analysis report from the Dashboard.

4. Discovery Engine (Upload Your Data)

The Discovery Engine (/discovery-engine) lets you upload your own CSV data and run AR(2) analysis on it. This works with any time-series data — gene expression, wearable sensor data, clinical measurements, etc.

CSV Format Requirements

Your file should have:

  • Column 1: Gene/variable name (text)
  • Columns 2+: Numeric measurements at sequential time points
  • Header row: Optional but recommended
  • Minimum: At least 8 time points for reliable AR(2) fitting

What You Get

  • Eigenvalue modulus |λ| for every row in your data
  • AR(2) coefficients (β₁, β₂) and R² goodness-of-fit
  • Edge case diagnostics (trend, sample size, stationarity)
  • Interactive charts with your data
  • Shareable link to your results
Note: More time points = more reliable results. With fewer than 8 time points, the AR(2) model has limited degrees of freedom and results should be interpreted cautiously. The diagnostics panel will flag this automatically.

5. Root-Space Geometry

Root-Space (/root-space) is one of the most powerful features. It maps every gene's AR(2) coefficients (β₁, β₂) onto a 2D plane, creating a "dynamical landscape" where position encodes behavior.

5.1 The Stationarity Triangle

The stationarity triangle defines the mathematically allowed region for stable AR(2) processes. Its boundaries are:

  • Top edge (β₂ = 1 − β₁): Oscillatory boundary — genes near here show strong rhythms
  • Bottom edge (β₂ = −1): Alternating sign boundary
  • Left/Right edges: Monotone decay/growth boundaries

Genes closer to the triangle's edges have stronger persistence (higher |λ|). Genes near the center have weaker persistence.

5.2 Overlays

Root-Space offers several interpretive overlays:

Waddington Landscape

Inspired by Waddington's epigenetic landscape metaphor. Valleys represent stable attractors (gene expression states), ridges represent barriers between states. Clock genes cluster in deep valleys; target genes on shallower slopes.

Phase Portrait

Visualizes the dynamical behavior at each point in coefficient space — showing whether expression at that position would oscillate, decay monotonically, or show alternating patterns.

Functional Geography

Colors the space by biological function, showing how different gene categories (immune, metabolic, structural, signaling) occupy different dynamical niches.

Void Annotation

Highlights regions of coefficient space where no genes are found ("dynamical voids"), suggesting these are biologically forbidden or selected against.

5.3 Drug Target Overlay

The Drug Target Overlay maps FDA-approved and investigational drug targets onto root-space positions. This reveals which drug targets are clock-gated — meaning their expression is governed by circadian rhythms and may respond differently depending on time of day.

What the Drug Target Chapter Tells Us

  • Position in root-space — Where each drug target gene sits in dynamical space. Genes near the oscillatory boundary are more clock-influenced.
  • Clock-gated percentage — What fraction of drug targets show self-reinforcing and oscillatory poles (meaning they have strong circadian regulation).
  • Chronotherapy candidates — Drug targets with high eigenvalue persistence are candidates for time-of-day dosing optimization. If a target gene's expression varies 2–5x over 24 hours, giving the drug at peak target expression could improve efficacy.
  • Drug class clustering — Different drug classes (kinase inhibitors, nuclear receptors, etc.) cluster in different regions of root-space, revealing class-wide dynamical properties.
  • Cross-tissue consistency — The same drug target may be clock-gated in liver but not in lung, informing tissue-specific dosing strategies.
Key finding: A significant fraction of FDA-approved drug targets show clock-gated dynamics, meaning their efficacy could potentially be improved through chronotherapy (timed dosing). This is consistent with growing clinical evidence that drug timing affects outcomes.

6. Validation Suite

The Validation Suite (/validation-suite) aggregates multiple independent lines of evidence confirming that AR(2) eigenvalues capture real biology, not statistical artifacts.

What's Validated

  • Multi-species hierarchy — Clock > target preserved in mouse, human, baboon, and plant
  • AR(1) vs AR(2) vs AR(3) comparison — Information criteria (AIC/BIC) confirm AR(2) is the preferred model order across datasets
  • Eigenvalue independence — |λ| is independent of mean expression level, ruling out abundance as a confound
  • Downstream prediction — Eigenvalues predict functional categories beyond what expression level alone captures

7. ODE Model Zoo

The ODE Model Zoo (/model-zoo) validates AR(2) eigenvalue predictions against five canonical ODE models from biology:

  • Goodwin Oscillator — Classic negative feedback loop
  • Van der Pol — Limit cycle oscillator
  • Repressilator — Synthetic gene circuit
  • Lotka-Volterra — Predator-prey dynamics
  • Toggle Switch — Bistable gene circuit

For each model, synthetic time-series are generated, AR(2) is fitted, and eigenvalues are compared to known analytical predictions. This demonstrates that AR(2) accurately recovers the dynamical properties of systems with known ground truth.

8. Genome-Wide Analysis

The Genome-Wide page (/genome-wide) runs AR(2) on every gene in a dataset (typically 10,000–20,000 genes), not just clock and target genes.

Key Features

  • Distribution histogram — Shows the genome-wide eigenvalue distribution. Clock genes sit in the right tail.
  • Percentile ranking — Where clock and target genes rank among all genes
  • Downloadable CSV — Per-gene eigenvalue table for all genes in the dataset
  • Functional enrichment — What biological functions are enriched at high vs low eigenvalues
Tip: The genome-wide distribution typically shows that clock genes are in the top 5–15% of all genes by eigenvalue, and the hierarchy emerges naturally without any prior labeling.

9. Cross-Context Validation

Cross-Context Validation (/cross-context-validation) combines species comparison and cross-tissue analysis in one view.

What It Tests

  • Cross-species — Does the clock > target hierarchy hold in mouse, human, baboon, and plant?
  • Cross-tissue — Does the hierarchy hold across liver, lung, SCN, kidney, heart, and other tissues?
  • Permutation tests — Statistical significance of the gap (p-values from 2,000+ permutations)
  • Bootstrap confidence intervals — 95% CIs on eigenvalue means for each category
Key result: The clock > target hierarchy is preserved across all species and tissues tested, with p < 0.001 in most cases. This universality argues it reflects fundamental circadian architecture, not tissue-specific artifacts.

10. Robustness Suite

The Robustness Suite (/robustness-suite) runs a battery of tests to ensure findings are not artifacts:

  • Bootstrap confidence intervals — Resample time points 1,000 times to get CIs on eigenvalues
  • Permutation tests — Shuffle clock/target labels to test if the gap is significant
  • Sub-sampling recovery — Remove random time points and check if hierarchy survives
  • Rolling window analysis — Test eigenvalue stability across sliding windows of time
  • Stationarity checks — ADF tests to ensure time series are stationary (required for AR modeling)
  • Population-level CV stability — Coefficient of variation across datasets

The Circadian Health Score (/health-score) distills the full AR(2) analysis into a single 0–100 score with A–F letter grades.

The score combines multiple metrics: clock gene eigenvalue means, target gene eigenvalue means, the clock–target gap size, model fit quality, and consistency across genes. A score of 80–100 (Grade A) indicates robust circadian function; below 40 (Grade D/F) suggests significant disruption.

19.1 Proteomics Chapter

Chapter 5 extends AR(2) analysis to protein-level data from three peer-reviewed datasets:

  • Wang 2018 — Mouse liver nuclear proteome (442 proteins, 6 timepoints)
  • Robles 2014 — Mouse liver whole-cell proteome (3,072 proteins, 16 timepoints, SILAC quantification)
  • Johonnuson 2025 — SCN proteome (1,045 proteins, 6 timepoints)

Key finding: AR(2) eigenvalue analysis works at the protein level too — cycling proteins show higher persistence (|λ| 0.535) than non-cycling (0.519), validated against Robles Table S2 cycling annotations (JTK_CYCLE method, 186 proteins).

19.2 Gene-Protein Persistence Map

The Gene-Protein Persistence Map (/gene-protein-map) is a dedicated page comparing mRNA and protein eigenvalues gene-by-gene across 2,718 matched genes.

Novel Finding

The mRNA-protein eigenvalue correlation is essentially zero (Pearson r = −0.006). This means: a gene's temporal persistence at the mRNA level tells you almost nothing about its temporal persistence at the protein level. They operate independently.

This is a novel measurement — previous studies examined mRNA-protein abundance correlation (which is weak, r ≈ 0.4–0.6). But persistence correlation is even weaker, suggesting that temporal memory at each layer is controlled by different mechanisms (transcriptional regulation for mRNA, post-translational modification for protein).

25. Cross-Metric Independence

The Cross-Metric Independence page (/cross-metric-independence) answers a fundamental validation question: is the AR(2) eigenvalue |λ| actually measuring something new, or is it just repackaging existing metrics like network centrality, oscillation amplitude, or chromatin accessibility?

The page compares |λ| against curated reference values from STRING v12.0 (network degree), cosinor analysis (oscillation amplitude), and ENCODE/Roadmap (H3K4me3 chromatin marks) for the 53-gene clock and target panel measured across multiple tissues.

Scatter Plots & Correlations

The first four tabs show scatter plots of |λ| vs each external metric:

  • Network Centrality — |λ| vs STRING protein interaction degree. Weak correlation (ρ ≈ −0.29) confirms persistence is not a proxy for network connectivity.
  • Cosinor Amplitude — |λ| vs 24-hour oscillation amplitude. Moderate correlation (ρ ≈ 0.69) is expected since stronger oscillations require more persistence, but the relationship is far from 1.0.
  • Chromatin State — |λ| vs H3K4me3 active chromatin marks. Very weak correlation (ρ ≈ 0.08) confirms persistence is independent of epigenetic accessibility.
  • Functional Categories — Breakdown by Gene Ontology and KEGG pathway annotations showing how different functional groups distribute across eigenvalue space.

Gene selection: Use the search box at the top of the page to select specific genes (e.g., Per2, Myc, Wee1). Selected genes are highlighted as larger cyan dots across all scatter plots, and a side-by-side metric profile table appears showing |λ|, network degree, amplitude, and cross-tissue CV for each selected gene. This makes it easy to compare a clock gene against a target gene across all metrics simultaneously.

Root-Space Mapping Tab

The Root-Space Mapping tab plots genes in AR(2) coefficient space (φ₁, φ₂) — the same coordinate system used in the Root-Space Geometry and Waddington Landscape pages. Each dot represents a gene at its fitted position. Four color-by options let you overlay external metrics on top of this landscape:

  • Gene Type — Clock genes (blue) vs target genes (amber). Clock genes cluster in the high-persistence region.
  • Network Degree — Green (<100 connections), yellow (100–200), red (200+). If colors are scattered rather than zoned, it confirms root-space position is independent of network topology.
  • Amplitude — Gray (<20), amber (20–100), red (100+). Some spatial structure is expected due to the moderate |λ|–amplitude correlation, but it should not form clean gradient bands.
  • Chromatin — Purple (Active) vs cyan (Poised). Mixed distribution confirms independence from epigenetic state.

Below the scatter plot, correlation statistics show Spearman ρ between root-space polar coordinates (radius |r| and angle θ) and external metrics. Key results: radius vs network degree ρ ≈ −0.29 (weak), radius vs amplitude ρ ≈ 0.69 (moderate), and angle vs all metrics ρ ≈ 0 (independent). This demonstrates that the root-space landscape cannot be reconstructed from existing metrics — it captures a genuinely independent biological property: temporal persistence dynamics.

Why this matters: If someone argues that |λ| is redundant with network centrality or chromatin state, this page provides the counter-evidence. The landscape topology is structurally different from what other metrics predict, confirming AR(2) reveals an independent analytical dimension.

Cross-Tissue Conservation

The Cross-Tissue Conservation tab compares how stable each gene's eigenvalue is across different tissues (liver, kidney, heart, lung). Clock genes show lower coefficient of variation (CV ≈ 0.12) than target genes (CV ≈ 0.24), consistent with clock genes being more tightly regulated across the body. The conservation ratio (clock CV / target CV ≈ 0.53) provides a quantitative measure of this hierarchy.

29. Glossary of Key Terms

TermDefinition
AR(2)Second-order autoregressive model. Predicts current value from two previous values.
|λ| (Eigenvalue Modulus)A number between 0 and 1 measuring how strongly a gene's expression persists over time. Higher = more persistent.
β₁, β₂The two AR(2) coefficients. Together they define a gene's "dynamical signature" and position in root-space.
R²Goodness-of-fit. How well the AR(2) model explains the gene's expression pattern. Higher = better fit.
Clock GeneA gene in the core circadian feedback loop (Per2, Arntl, Clock, Cry1, etc.). Self-reinforcing, high |λ|.
Target GeneA gene downstream of the clock — it responds to circadian signals but doesn't sustain its own rhythm. Lower |λ|.
Root-SpaceThe 2D plane of (β₁, β₂) where each gene's dynamical type is visible from its position.
Stationarity TriangleThe mathematical boundary in root-space within which AR(2) processes are stable.
Clock-GatedA gene whose dynamics are governed by circadian rhythms — its AR(2) coefficients place it in the oscillatory region of root-space.
ChronotherapyOptimizing drug dosing timing based on circadian biology. Clock-gated drug targets are chronotherapy candidates.
Permutation TestStatistical test that shuffles labels (e.g., clock vs target) many times to assess if the observed difference is significant.
Bootstrap CIConfidence interval estimated by resampling the data with replacement, typically 1,000 times.
ADF TestAugmented Dickey-Fuller test for stationarity — checks whether a time series has a stable mean and variance.
AIC/BICInformation criteria for model selection. Lower = better. Used to compare AR(1), AR(2), AR(3) fits.
Edge Case DiagnosticsAutomated checks for data quality issues: trends, short series, nonlinearity, boundary proximity.
SILACStable Isotope Labeling with Amino acids in Cell culture — a proteomics quantification method used in Robles 2014.
JTK_CYCLEA statistical method for detecting rhythmic patterns in time-series data, used to identify cycling proteins.
GEOGene Expression Omnibus — NCBI's public repository of gene expression datasets.
Waddington LandscapeA metaphor from developmental biology: a marble rolling down hills represents cell fate decisions. AR(2) root-space provides a quantitative version.

30. Frequently Asked Questions

Q: Do I need to understand the math to use this platform?

No. The platform handles all computation automatically. The key concept is simple: |λ| near 1 means strong temporal persistence, near 0 means weak. Clock genes are consistently higher than target genes. Everything else is detail.

Q: What data format does the Discovery Engine accept?

Standard CSV files with gene names in the first column and numeric expression values in subsequent columns (one column per time point). At least 8 time points recommended.

Q: Why AR(2) and not AR(1) or AR(3)?

AR(1) can only capture monotone decay — it misses oscillations entirely. AR(3) adds a parameter without significantly improving model fit. AR(2) is the minimum model that captures both persistence and oscillation, and information criteria (AIC/BIC) confirm it's the preferred order across dozens of datasets.

Q: Can I use this with non-circadian data?

Yes. The AR(2) framework works on any time-series data. The eigenvalue captures temporal persistence regardless of whether the underlying process is circadian, metabolic, developmental, or something else. The clock/target labels are specific to circadian analysis, but the Discovery Engine accepts any data.

Q: How do I know if my results are reliable?

Check the Edge Case Diagnostics panel (visible on the Dashboard) and the Robustness Suite. Key indicators: R² > 0.1 (model explains something), no trend warnings, at least 10 time points, and ADF stationarity check passes.

Q: What does "clock-gated" mean for drug targets?

A clock-gated drug target is a gene whose expression dynamics are significantly influenced by circadian rhythms. If the drug target's expression peaks at a specific time of day, the drug may be more effective when dosed at that time (chronotherapy).

Q: What does the Gene-Protein Map show that's new?

It shows that temporal persistence at the mRNA level is essentially uncorrelated (r ≈ 0) with temporal persistence at the protein level. Previous studies showed weak abundance correlation, but persistence independence is a new finding — it means mRNA and protein dynamics are controlled by different mechanisms.

Q: Can I download my results?

Yes. Most pages offer CSV downloads of per-gene eigenvalue tables. The Dashboard also supports chart export as PNG/SVG and printable analysis reports.

Q: How many genes can I analyze at once?

The genome-wide analysis runs on 10,000–20,000 genes simultaneously. The Discovery Engine handles CSV files with thousands of rows. The platform is designed for large-scale analysis.

Q: What species are supported?

Pre-loaded datasets cover mouse, human, baboon, and Arabidopsis (plant). The Discovery Engine accepts data from any species as long as it's in CSV format.

31. Troubleshooting

Problem: Analysis shows all eigenvalues near zero

This usually means the time series is too short or has no temporal structure. Check that you have at least 8 time points and that the data shows some variation over time. Flat or random data will produce low eigenvalues (which is correct — the data truly has no persistence).

Problem: Edge case diagnostic flags "trend"

A linear trend in the data can inflate eigenvalues. Consider detrending your data before upload, or interpret results cautiously. The platform flags this automatically so you're aware.

Problem: CSV upload fails

Ensure your CSV has: (1) gene names in column 1, (2) numeric values in all other columns, (3) no empty rows or extra formatting. Remove any header rows that contain non-numeric data beyond the first row.

Problem: No clock genes found in dataset

The platform searches for standard gene names (Per2, Arntl, Clock, etc.). If your dataset uses different naming (e.g., BMAL1 instead of Arntl), the genes may not be recognized automatically. You can manually select genes from the dropdown.

Problem: Page loads slowly

Large analyses (genome-wide, cross-tissue) compute thousands of AR(2) models. Allow 10–30 seconds for these pages to load. Results are cached after the first load.
