1. Introduction & Core Concepts

What is the PAR(2) Discovery Engine?

The PAR(2) Discovery Engine is a statistical platform for analyzing gene expression time-series data. Its central idea: fit a second-order autoregressive model — AR(2) — to each gene's expression over time, then extract the eigenvalue modulus |λ| as a single number that captures how strongly that gene's expression "persists" from one time point to the next.

The platform ships with 38+ pre-loaded datasets from public repositories (NCBI GEO), spanning mouse, human, baboon, yeast, and Arabidopsis tissues. You can also upload your own data — gene expression matrices, wearable device exports, or any generic CSV with time-series columns — and get instant AR(2) analysis with quality diagnostics.

The core question: Instead of asking "Is this gene rhythmic?" (yes/no), AR(2) asks "How persistent is this gene's temporal pattern?" — a continuous measurement that captures both oscillation strength and memory.

What is AR(2)?

An autoregressive model of order 2 means: the expression value at time t depends on the values at times t−1 and t−2, plus noise. Mathematically: x(t) = φ₁ · x(t−1) + φ₂ · x(t−2) + ε

The two coefficients φ₁ and φ₂ define the gene's "dynamical signature." From these, we compute the eigenvalue modulus |λ|, which ranges from 0 to 1:

|λ| ValueInterpretationBiological Example
0.0 – 0.3Weak persistence — expression decays quicklyNoise-driven, rapidly turning over genes
0.3 – 0.6Moderate persistence — some temporal memoryTarget genes responding to signals
0.6 – 0.8Strong persistence — robust temporal patternClock-controlled output genes
0.8 – 1.0Very strong persistence — self-sustaining oscillationCore clock genes (Per2, Arntl, Clock)

Complex vs Real Eigenvalues

The AR(2) characteristic equation can yield either real or complex conjugate eigenvalues. Complex eigenvalues indicate oscillatory dynamics — the gene's expression pattern naturally oscillates. Real eigenvalues indicate monotone decay or growth without oscillation. The engine reports this distinction for every analyzed gene, along with the implied period (in hours) when eigenvalues are complex.

The Clock > Target Hierarchy

The central finding of the PAR(2) framework is the clock > target eigenvalue hierarchy: core circadian clock genes (Per2, Arntl, Clock, Cry1, etc.) consistently show higher |λ| values than their downstream target genes (Myc, Ccnd1, Wee1, etc.) across tissues, species, and conditions.

This is not just a statistical observation — it reflects biology. Clock genes form self-reinforcing transcription-translation feedback loops, giving them stronger temporal memory. Target genes respond to clock signals but lack their own sustaining loops.

2. Quick Start (5-Minute Tour)

Follow these steps to get oriented in 5 minutes:

Step 1: Open the Dashboard

The home page (/) shows summary statistics and a list of pre-loaded datasets. You'll see 38+ datasets spanning mouse, human, baboon, yeast, and Arabidopsis tissues. No upload or account required.

Step 2: Select a Dataset & Run Analysis

Pick any dataset (start with Mouse Liver (GSE11923) — it has the most time points). Click "Run Analysis". The engine computes AR(2) eigenvalues for all clock and target genes in seconds.

Step 3: Read the Results

You'll see eigenvalue bar charts, time-series plots, and the clock-vs-target comparison. The gap between clock and target eigenvalues is the key metric — a larger gap means stronger circadian hierarchy.

Step 4: Explore Root-Space

Navigate to /root-space to see where every gene lands in coefficient space. Try the Waddington landscape, phase portrait, functional geography, and drug target overlays.

Step 5: Try Your Own Data

Go to /discovery-engine and upload a CSV file with time-series data. The engine will compute AR(2) eigenvalues with full diagnostics. For gene expression matrices with CT/ZT column headers, you'll get per-gene analysis with Gearbox hierarchy testing.

Step 6: Share Your Results

After running an analysis in the Discovery Engine, click "Share Results" to generate a unique URL. Anyone with the link can view your full analysis at /shared/:id.

Tip: Use the navigation menu (top-left hamburger icon) to jump between all pages. Each page has a "How To" helper that explains what it does and how to use it.

3. What You Can and Cannot Do

This section provides a clear summary of the platform's capabilities and limitations so you know exactly what to expect before you start.

What You CAN Do

  • Upload CSV/TSV/TXT gene expression matrices and get per-gene AR(2) eigenvalue analysis with automatic clock/target classification
  • Upload wearable device data (heart rate, HRV, steps, sleep stages, CGM glucose) for AR(2) persistence analysis
  • Upload any generic time-series CSV data — lab measurements, environmental sensors, custom signals
  • Share results via unique shareable links — Discovery Engine creates a permanent URL viewable on the Shared Analysis page
  • Download HTML reports from the Disease Screen, Robustness Suite, and Wearable Analysis pages
  • Download this user guide as an HTML file (save as PDF from your browser's Print dialog)
  • Browse 38+ pre-loaded datasets without uploading anything — mouse, human, baboon, yeast, Arabidopsis
  • Compare before/after conditions by uploading two CSV files on the Before/After page
  • Test custom gene lists against genome-wide eigenvalue distributions on the Gene Set Hypothesis Tester
  • Explore individual genes across all datasets via the Gene Explorer
  • Download per-gene eigenvalue CSV tables from the Genome-Wide Analysis page
  • View 3D interactive convergence maps showing how independent research streams converge with PAR(2) findings
  • Cross-reference PAR(2) findings against STRING protein interaction data, cosinor amplitude, and ENCODE chromatin metrics

What You CANNOT Do

  • Feed Discovery Engine results into other pages — each analysis page (Root-Space, Robustness, Disease Screen, etc.) works independently with its own pre-loaded datasets. There is no pipeline connecting uploaded data to other pages.
  • Save analyses for later within the app — your analysis is ephemeral unless you create a share link. Close the browser tab and your results are gone.
  • Modify pre-loaded datasets — the 38+ embedded datasets are read-only. You can select which genes to analyze but cannot edit the underlying data.
  • Run analysis on images, PDFs, or non-tabular files — only plain-text CSV, TSV, and TXT files are accepted.
  • Upload files larger than ~50 MB — the engine enforces a file size limit for server stability.
  • Get real-time streaming results — analysis runs to completion and then displays all results at once.
  • Edit or update a shared analysis link once created — each share link is a permanent, immutable snapshot.
Important: The platform is a research tool, not a clinical diagnostic. All findings require independent experimental validation before any clinical application.

4.1 Selecting Datasets

The platform includes 38+ datasets from public repositories (primarily NCBI GEO). Each dataset card shows:

  • Species — Mouse, Human, Baboon, Yeast, Arabidopsis

Select clock and target genes from the gene lists, then click "Run Analysis". The engine fits AR(2) models to every selected gene and returns eigenvalues within seconds.

You can customize which genes to analyze — the defaults include the core clock genes (Per2, Arntl, Clock, Cry1, Cry2, Nr1d1, Nr1d2, Per1) and representative target genes (Myc, Ccnd1, Wee1, Cdkn1a, etc.).

4.3 Reading Results

  • Time-Series Plot — Raw expression data over time for selected genes, showing the oscillatory patterns.
  • AR(2) Coefficients — The raw φ₁ and φ₂ values and model fit (R²).
  • Stability Triangle Diagram — Each gene plotted in the stationarity triangle, showing its dynamical regime.
  • Unit Circle Diagram — Eigenvalue roots plotted relative to the unit circle boundary.
  • Edge Case Diagnostics — Reliability checks for trend contamination, sample size, nonlinearity, and boundary proximity.
Tip: You can download results as CSV, export charts as images, and generate a printable analysis report from the Dashboard.

The Discovery Engine (/discovery-engine) lets you upload your own CSV data and run AR(2) analysis on it. The engine auto-detects the data format and adapts its analysis accordingly. This works with gene expression matrices, wearable sensor data, lab measurements, or any generic time-series CSV.

Data privacy: Your uploaded file is processed in memory and is not stored on the server. Only analysis results (gene names, eigenvalues, statistics) are saved to enable sharing features. The raw data itself is never persisted.

5.2 Gene Expression Matrix Mode

When the Discovery Engine detects that your CSV has column headers containing CT (circadian time) or ZT (zeitgeber time) patterns, it activates Gene Expression Matrix mode. This is the most powerful analysis mode, designed specifically for circadian gene expression experiments.

How It Works

  1. Format detection: The engine scans column headers for CT/ZT patterns (e.g., CT0, CT4, CT8, ZT0, ZT6, ZT12). If found, it treats each row as a gene and each column as a time point.
  2. Per-gene AR(2) fitting: An independent AR(2) model is fitted to each gene's expression values across all time points. This yields φ₁, φ₂, |λ|, R², and eigenvalue type (complex/real) for every gene.
  3. Clock/target classification: Genes are automatically classified using a built-in 15-gene clock gene list: Per1, Per2, Per3, Cry1, Cry2, Clock, Arntl (Bmal1), Nr1d1, Nr1d2, Rora, Rorc, Dbp, Tef, Hlf, Nfil3, and Bhlhe40. All other genes are classified as "target."
  4. Gearbox Analysis: The engine computes the mean eigenvalue for clock genes and the mean eigenvalue for target genes, then reports the hierarchy gap (Δ|λ|) and whether the expected clock > target hierarchy is preserved or reversed.
  5. Ranking: The Top 20 genes by eigenvalue and Bottom 10 genes by eigenvalue are displayed in ranked cards.

Gene Card Details

Each gene in the results shows a detailed card with:

  • Gene name — as it appears in your CSV's first column
  • Gene type — Clock or Target
  • |λ| (eigenvalue modulus) — the persistence score, 0 to 1
  • φ₁ and φ₂ — the AR(2) coefficients
  • R² — goodness-of-fit of the AR(2) model
  • Eigenvalue type — Complex (oscillatory) or Real (monotone)
  • Implied period — when eigenvalues are complex, the implied oscillation period in hours
  • Stability category — color-coded classification (e.g., "Stationary," "Near boundary," "Explosive")

CSV Format for Gene Expression Matrices

Minimum time points: At least 6 time points are needed for AR(2) fitting (the model uses 2 lags, leaving n−2 residuals). With fewer than 8 time points, the AR(2) model has limited degrees of freedom and results should be interpreted cautiously. The diagnostics panel will flag this automatically.

5.3 Wearable & Generic CSV Mode

For non-gene-expression data (wearable exports, lab measurements, environmental sensors), the engine treats each numeric column as a separate "channel" and fits AR(2) independently to each. You get the same eigenvalue, coefficient, R², and diagnostic information per channel.

The engine also provides sample data generators for testing: synthetic glucose data, HRV data, and generic time-series signals. These are useful for understanding the output before analyzing your own data.

6. Root-Space Geometry

Root-Space (/root-space) is one of the most powerful features. It maps every gene's AR(2) coefficients (φ₁, φ₂) onto a 2D plane, creating a "dynamical landscape" where position encodes behavior.

6.1 The Stationarity Triangle

The stationarity triangle defines the mathematically allowed region for stable AR(2) processes. Its boundaries are:

  • Top edge (φ₂ = 1 − φ₁): Oscillatory boundary — genes near here show strong rhythms
  • Bottom edge (φ₂ = −1): Alternating sign boundary
  • Left/Right edges: Monotone decay/growth boundaries

Genes closer to the triangle's edges have stronger persistence (higher |λ|). Genes near the center have weaker persistence. Genes outside the triangle would be explosively unstable.

6.2 Overlays

Root-Space offers several interpretive overlays:

Waddington Landscape

Inspired by Waddington's epigenetic landscape metaphor. Valleys represent stable attractors (gene expression states), ridges represent barriers between states. Clock genes cluster in deep valleys; target genes on shallower slopes.

Phase Portrait

Visualizes the dynamical behavior at each point in coefficient space — showing whether expression at that position would oscillate, decay monotonically, or show alternating patterns.

Functional Geography

Colors the space by biological function, showing how different gene categories (immune, metabolic, structural, signaling) occupy different dynamical niches.

Void Annotation

Highlights regions of coefficient space where no genes are found ("dynamical voids"), suggesting these are biologically forbidden or selected against.

6.3 Drug Target Overlay

The Drug Target Overlay maps FDA-approved and investigational drug targets onto root-space positions. This reveals which drug targets are clock-gated — meaning their expression is governed by circadian rhythms and may respond differently depending on time of day.

What the Drug Target Chapter Tells Us

  • Position in root-space — Where each drug target gene sits in dynamical space. Genes near the oscillatory boundary are more clock-influenced.
  • Clock-gated percentage — What fraction of drug targets show self-reinforcing and oscillatory poles (meaning they have strong circadian regulation).
  • Chronotherapy candidates — Drug targets with high eigenvalue persistence are candidates for time-of-day dosing optimization.
  • Drug class clustering — Different drug classes (kinase inhibitors, nuclear receptors, etc.) cluster in different regions of root-space, revealing class-wide dynamical properties.
  • Cross-tissue consistency — The same drug target may be clock-gated in liver but not in lung, informing tissue-specific dosing strategies.
Key finding: A significant fraction of FDA-approved drug targets show clock-gated dynamics, meaning their efficacy could potentially be improved through chronotherapy (timed dosing). This is consistent with growing clinical evidence that drug timing affects outcomes.

7. Validation Suite

The Validation Suite (/validation-suite) aggregates multiple independent lines of evidence confirming that AR(2) eigenvalues capture real biology, not statistical artifacts.

What's Validated

  • Multi-species hierarchy — Clock > target preserved in mouse, human, baboon, and plant
  • AR(1) vs AR(2) vs AR(3) comparison — Information criteria (AIC/BIC) confirm AR(2) is the preferred model order across datasets
  • Eigenvalue independence — |λ| is independent of mean expression level, ruling out abundance as a confound
  • Downstream prediction — Eigenvalues predict functional categories beyond what expression level alone captures
  • ODE ground truth — AR(2) eigenvalues recover known dynamics from five canonical ODE models
  • Permutation null distributions — Shuffled labels produce no hierarchy, confirming the signal is real

8. Literature Validation & Falsification

The Literature Validation page (/literature-validation) provides the strongest external validation of the PAR(2) framework. It cross-references blind PAR(2) eigenvalue discoveries against 59 curated literature-validated circadian genes from published experimental studies.

How It Works

  1. The PAR(2) engine ranks all genes by eigenvalue without any prior knowledge of which genes are known circadian regulators.
  2. The top-ranked genes are compared against the curated list of 59 experimentally validated circadian genes from published literature.
  3. Overlap, enrichment p-values, and precision/recall metrics are computed.

Falsification Test

The page also includes a built-in falsification test: the Arntl predictor is compared against housekeeping gene controls and randomly sampled gene sets. If the AR(2) framework were merely capturing noise or abundance effects, the housekeeping and random controls would perform equally well. They do not — the circadian gene set is significantly enriched at high eigenvalues compared to both controls.

9. ODE Model Zoo

The ODE Model Zoo (/model-zoo) validates AR(2) eigenvalue predictions against five canonical ODE models from biology:

  • Goodwin Oscillator — Classic negative feedback loop
  • Van der Pol — Limit cycle oscillator
  • Repressilator — Synthetic gene circuit
  • Lotka-Volterra — Predator-prey dynamics
  • Toggle Switch — Bistable gene circuit

For each model, synthetic time-series are generated, AR(2) is fitted, and eigenvalues are compared to known analytical predictions. This demonstrates that AR(2) accurately recovers the dynamical properties of systems with known ground truth.

10. Genome-Wide Analysis

The Genome-Wide page (/genome-wide) runs AR(2) on every gene in a dataset (typically 10,000–20,000 genes), not just clock and target genes.

Key Features

  • Distribution histogram — Shows the genome-wide eigenvalue distribution. Clock genes sit in the right tail.
  • Percentile ranking — Where clock and target genes rank among all genes
  • Downloadable CSV — Per-gene eigenvalue table for all genes in the dataset (one-click download)
  • Functional enrichment — What biological functions are enriched at high vs low eigenvalues
Tip: The genome-wide distribution typically shows that clock genes are in the top 5–15% of all genes by eigenvalue, and the hierarchy emerges naturally without any prior labeling.

11. Genome-Wide Coupling Scan

The Genome-Wide Coupling Scan (/genome-wide-coupling) extends beyond single-gene eigenvalue analysis to test all ~21,000 genes for statistical coupling to the core circadian clock using AR(2)+exogenous models.

How It Works

  1. For each gene, an AR(2) model is fitted with and without an exogenous clock gene predictor (typically Arntl/BMAL1).
  2. A likelihood ratio test determines whether including the clock gene significantly improves the fit.
  3. Benjamini-Hochberg FDR correction is applied across all ~21,000 tests to control the false discovery rate at 5%.
  4. Genes passing FDR are classified as "coupled" to the circadian clock.
  5. Hypergeometric pathway enrichment tests whether coupled genes are enriched in specific biological pathways (KEGG, Reactome, GO).

Key Outputs

  • Total number of coupled genes (typically 2,000–5,000 depending on tissue)
  • Coupling strength distribution
  • Top enriched pathways among coupled genes
  • Volcano plot of coupling effect size vs statistical significance

12. Cross-Context Validation

Cross-Context Validation (/cross-context-validation) combines species comparison and cross-tissue analysis in one view.

What It Tests

  • Cross-species — Does the clock > target hierarchy hold in mouse, human, baboon, and plant?
  • Cross-tissue — Does the hierarchy hold across liver, lung, SCN, kidney, heart, and other tissues?
  • Permutation tests — Statistical significance of the gap (p-values from 2,000+ permutations)
  • Bootstrap confidence intervals — 95% CIs on eigenvalue means for each category
Key result: The clock > target hierarchy is preserved across all species and tissues tested, with p < 0.001 in most cases. This universality argues it reflects fundamental circadian architecture, not tissue-specific artifacts.

13. Robustness Suite

The Robustness Suite (/robustness-suite) runs a battery of tests to ensure findings are not artifacts. It includes a Download Report button that generates a standalone HTML file summarizing all robustness tests.

  • Bootstrap confidence intervals — Resample time points 1,000 times to get CIs on eigenvalues
  • Permutation tests — Shuffle clock/target labels to test if the gap is significant
  • Sub-sampling recovery — Remove random time points and check if hierarchy survives
  • Rolling window analysis — Test eigenvalue stability across sliding windows of time
  • Stationarity checks — ADF tests to ensure time series are stationary (required for AR modeling)
  • Population-level CV stability — Coefficient of variation across datasets

15. Rolling Window Stability

The Rolling Window Stability page (/rolling-window) tests whether AR(2) eigenvalue estimates are stable across different sub-windows of time within a dataset.

How It Works

A sliding window of fixed width (e.g., 12 or 24 time points) moves across the full time series. At each position, AR(2) is re-fitted and eigenvalues are recomputed. If the eigenvalue is stable (low variance across windows), the measurement is robust. If eigenvalues fluctuate widely, the gene may be non-stationary or affected by transient perturbations.

What You See

  • Per-gene eigenvalue trajectory across windows
  • Stability coefficient (CV of eigenvalue across windows)
  • Clock vs target stability comparison

19. Cancer Browser

The Cancer Browser (/cancer-browser) applies AR(2) analysis to cancer gene expression datasets, comparing tumor vs. normal tissue eigenvalue profiles. It examines how oncogenic mutations and tumor microenvironment alter circadian temporal dynamics.

What It Shows

  • Tumor vs normal eigenvalue comparison — Side-by-side eigenvalue profiles for matched tumor and normal tissue datasets
  • Clock gene disruption scores — How much each core clock gene's persistence changes in the tumor state
  • Oncogene eigenvalue tracking — How oncogenes (Myc, Ccnd1, Cdk4) shift in root-space under transformation
  • APC mutation effects — Specific analysis of Wnt pathway mutations on circadian hierarchy
Key finding: In multiple cancer types, the clock > target hierarchy is weakened or reversed, consistent with the hypothesis that circadian disruption is a hallmark of cancer progression. Oncogenes tend to gain persistence while clock genes lose it.

20. Circadian Health Score

The Circadian Health Score (/health-score) distills the full AR(2) analysis into a single 0–100 score with A–F letter grades.

How It Works

The score combines multiple metrics: clock gene eigenvalue means, target gene eigenvalue means, the clock–target gap size, model fit quality, and consistency across genes. A score of 80–100 (Grade A) indicates robust circadian function; below 40 (Grade D/F) suggests significant disruption.

Tip: Compare health scores across datasets to see which tissues, species, or conditions show the strongest circadian signatures.

21. Most Volatile Genes

The Most Volatile Genes page (/volatile-genes) ranks genes by how much their eigenvalue varies across datasets.

A gene with high volatility has very different persistence in different tissues or conditions. This is biologically interesting — it suggests tissue-specific regulation of temporal dynamics. Low-volatility genes maintain consistent dynamics everywhere; high-volatility genes are context-dependent.

22. Gene Set Hypothesis Tester

The Gene Set Hypothesis Tester (/gene-set-tester) lets you test any custom gene list against the genome-wide eigenvalue distribution.

How to Use It

  1. Enter a list of gene names (one per line or comma-separated)
  2. Select a dataset to test against
  3. The engine runs 5,000 permutations to test whether your gene set has significantly higher or lower eigenvalues than expected by chance
  4. Results include p-value, effect size (Cohen's d), and a distribution plot
Example: Paste a list of inflammation-related genes to test whether inflammatory pathways show different temporal persistence than the genome background.

24.2 Gene-Protein Persistence Map

The Gene-Protein Persistence Map (/gene-protein-map) is a dedicated page comparing mRNA and protein eigenvalues gene-by-gene across 2,718 matched genes.

Novel Finding

The mRNA-protein eigenvalue correlation is essentially zero (Pearson r = −0.006). This means: a gene's temporal persistence at the mRNA level tells you almost nothing about its temporal persistence at the protein level. They operate independently.

This is a novel measurement — previous studies examined mRNA-protein abundance correlation (which is weak, r ≈ 0.4–0.6). But persistence correlation is even weaker, suggesting that temporal memory at each layer is controlled by different mechanisms (transcriptional regulation for mRNA, post-translational modification for protein).

32. Phase-Gating Analysis

The Phase-Gating Analysis page (/phase-gating) implements the phase-dependent PAR(2) methodology described in Paper E.

Core Concept

Standard AR(2) treats coefficients as constant over the entire time series. Phase-gating asks: do the AR(2) coefficients change depending on the circadian phase? For example, a gene might have different persistence during subjective day vs subjective night.

The methodology divides each 24-hour cycle into phase bins and fits separate AR(2) models within each. Genes whose coefficients change significantly across phases are "phase-gated" — their temporal dynamics are circadian-phase-dependent. This reveals a layer of regulation invisible to standard AR(2).

39. Glossary of Key Terms

TermDefinition
AR(2)Second-order autoregressive model. Predicts current value from two previous values: x(t) = φ₁·x(t−1) + φ₂·x(t−2) + ε.
|λ| (Eigenvalue Modulus)A number between 0 and 1 measuring how strongly a gene's expression persists over time. Higher = more persistent. Computed from the characteristic roots of the AR(2) polynomial.
φ₁, φ₂The two AR(2) coefficients. Together they define a gene's "dynamical signature" and position in root-space.
R²Goodness-of-fit. How well the AR(2) model explains the gene's expression pattern. Higher = better fit.
Clock GeneA gene in the core circadian feedback loop (Per2, Arntl, Clock, Cry1, etc.). Self-reinforcing, high |λ|.
Target GeneA gene downstream of the clock — it responds to circadian signals but doesn't sustain its own rhythm. Lower |λ|.
Gene Expression MatrixA tabular data format where rows are genes, columns are time points (labeled with CT or ZT headers), and values are expression levels. The Discovery Engine auto-detects this format for per-gene AR(2) analysis.
Gearbox AnalysisThe Discovery Engine's automated comparison of mean clock gene eigenvalue vs mean target gene eigenvalue. Reports whether the expected clock > target hierarchy is preserved or reversed, along with the gap magnitude and statistical confidence.
Hierarchy Gap (Δ|λ|)The difference between mean clock eigenvalue and mean target eigenvalue. A positive gap indicates preserved hierarchy; a negative gap indicates reversal. Larger gaps indicate stronger separation.
Root-SpaceThe 2D plane of (φ₁, φ₂) where each gene's dynamical type is visible from its position.
Stationarity TriangleThe mathematical boundary in root-space within which AR(2) processes are stable. Defined by three linear constraints on φ₁ and φ₂.
Clock-GatedA gene whose dynamics are governed by circadian rhythms — its AR(2) coefficients place it in the oscillatory region of root-space.
Phase-GatingA methodology where AR(2) coefficients are allowed to vary by circadian phase (e.g., day vs night). Reveals phase-dependent temporal dynamics invisible to standard constant-coefficient AR(2).
Coupling ScanGenome-wide test of statistical coupling between each gene and the core clock, using AR(2)+exogenous models. Identifies which genes are significantly clock-coupled after FDR correction.
BH-FDR CorrectionBenjamini-Hochberg False Discovery Rate correction. When testing thousands of genes simultaneously, this procedure controls the expected proportion of false positives at a chosen threshold (typically 5%).
Cohen's dA standardized effect size measure. d = (mean₁ − mean₂) / pooled SD. Values of 0.2, 0.5, and 0.8 are conventionally considered small, medium, and large effects.
Ljung-Box TestA statistical test of whether autocorrelations in a time series' residuals are different from zero. Used to verify that AR(2) model residuals are white noise (good fit). If the test rejects, the model may be inadequate.
Shareable LinkA permanent, immutable URL generated by the Discovery Engine that allows anyone to view an analysis snapshot. Format: /shared/:id. Stored in the platform database.
ChronotherapyOptimizing drug dosing timing based on circadian biology. Clock-gated drug targets are chronotherapy candidates.
Permutation TestStatistical test that shuffles labels (e.g., clock vs target) many times to assess if the observed difference is significant.
Bootstrap CIConfidence interval estimated by resampling the data with replacement, typically 1,000 times.
ADF TestAugmented Dickey-Fuller test for stationarity — checks whether a time series has a stable mean and variance.
AIC/BICInformation criteria for model selection. Lower = better. Used to compare AR(1), AR(2), AR(3) fits.
Edge Case DiagnosticsAutomated checks for data quality issues: trends, short series, nonlinearity, boundary proximity.
SILACStable Isotope Labeling with Amino acids in Cell culture — a proteomics quantification method used in Robles 2014.
JTK_CYCLEA statistical method for detecting rhythmic patterns in time-series data, used to identify cycling proteins.
GEOGene Expression Omnibus — NCBI's public repository of gene expression datasets.
Waddington LandscapeA metaphor from developmental biology: a marble rolling down hills represents cell fate decisions. AR(2) root-space provides a quantitative version.

40. Frequently Asked Questions

Q: Do I need to understand the math to use this platform?

No. The platform handles all mathematical details automatically. You upload data, click "Run Analysis," and read the results. The key output is |λ| — a number from 0 to 1 where higher means stronger temporal persistence. The glossary and this guide explain all terms.

Q: What data format does the Discovery Engine accept?

CSV, TSV, and TXT files containing plain-text tabular data. Each numeric column is treated as a time-series channel. For gene expression matrices, use genes as rows and time points (labeled CT or ZT) as columns. Excel .xlsx files, PDFs, and images are not accepted.

Q: Why AR(2) and not AR(1) or AR(3)?

AR(2) is the minimum order that captures oscillatory dynamics — AR(1) can only model exponential decay/growth. AR(3) adds complexity without significantly improving fit for circadian data (confirmed by AIC/BIC comparisons across 38+ datasets). AR(2) is the Goldilocks order for circadian gene expression.

Q: Can I use results from one page on another page?

No. Each analysis page works independently with its own pre-loaded datasets or uploaded data. The Discovery Engine does not feed results into Root-Space, Robustness Suite, Disease Screen, or any other page. The only cross-page connection is Discovery Engine → Shared Analysis (via share links).

Q: What happens to my shared link?

Shared links are stored permanently in the platform's database. Anyone with the full URL can view your analysis results at any time. The link is immutable — you cannot edit or delete a shared analysis once created. If you need updated results, run a new analysis and create a new share link.

Q: What file formats are supported?

CSV (comma-separated), TSV (tab-separated), and TXT (plain text) files only. All must contain tabular numeric data. Binary formats like Excel (.xlsx), images, PDFs, and compressed archives are rejected. Maximum file size is approximately 50 MB.

Q: How does the engine detect gene expression matrices?

The engine scans column headers for CT (circadian time) or ZT (zeitgeber time) patterns. If headers like "CT0", "CT4", "ZT18" are found, it treats the data as a gene expression matrix with genes as rows and time points as columns. Otherwise, it falls back to generic CSV mode where each numeric column is treated as a separate channel.

Q: What are the automatic data quality checks?

The engine runs six safeguards: (1) file type validation (CSV/TSV/TXT only), (2) duplicate gene detection with first-occurrence preservation, (3) constant expression gene exclusion (variance < 1e-10), (4) corrupted row exclusion (>50% missing values), (5) outlier detection (>10 SD flagging), and (6) NaN cleaning before AR(2) fitting. All checks produce color-coded warnings in the UI.

Q: Why does my hierarchy show "reversed"?

A "reversed" hierarchy means target genes have higher mean eigenvalues than clock genes in your data. This is not an error — it is a real finding that can occur in certain tissues, disease states, or experimental conditions. It suggests the normal circadian control hierarchy is disrupted. The Gearbox Analysis explicitly reports this so you can investigate further.

Q: Can I download my results?

Yes, in several ways: (1) Create a shareable link to view results anytime, (2) Download HTML reports from Disease Screen, Robustness Suite, and Wearable Analysis pages, (3) Download per-gene eigenvalue CSV tables from Genome-Wide Analysis, (4) Export individual charts as images from the Dashboard. Reports are standalone HTML files that can be converted to PDF via your browser's Print dialog.

Q: How many genes can I analyze at once?

The Discovery Engine can handle gene expression matrices with thousands of genes. The Genome-Wide Analysis page routinely processes 10,000–20,000+ genes. Performance depends on the number of time points (more time points = more computation per gene) but typical analyses complete within seconds to a few minutes.

Q: Is my data stored on your servers?

Your uploaded file is processed in memory and is NOT stored on the server. Only analysis results (gene names, eigenvalues, statistics, diagnostics) are saved to enable sharing features. The raw expression data is never persisted.

Q: What does "explosive" mean in the stability classification?

An "explosive" classification means |λ| > 1.0 — the AR(2) model predicts that the time series would grow without bound. This usually indicates non-stationarity (trends, step changes) in the data rather than true biological instability. The edge case diagnostics will flag this for further investigation.

Q: How do I interpret the Ljung-Box test result?

The Ljung-Box test checks whether the AR(2) model's residuals are random (white noise). A high p-value (> 0.05) means residuals are white noise — the model is adequate. A low p-value means residuals contain unexplained autocorrelation — a higher-order model or non-linear effects may be present. The result is shown per channel in the Discovery Engine results.

Q: Can I compare results across different tissues?

Yes. The Cross-Context Validation page compares eigenvalue hierarchies across tissues and species using the pre-loaded datasets. For uploaded data, you can run separate analyses in the Discovery Engine and compare the results manually or via share links. The Gene Explorer page also lets you track individual genes across all available datasets.

Q: What is the minimum number of time points needed?

A minimum of 6 time points is required for AR(2) fitting (the model uses 2 lags). However, for reliable results, 8 or more time points are recommended. With fewer than 8, the diagnostics panel will flag limited degrees of freedom as a warning.

41. Troubleshooting

File won't upload

Cause: The file is not in a supported format. The Discovery Engine accepts only CSV, TSV, and TXT files.

Fix: If you have an Excel .xlsx file, open it in Excel or Google Sheets and export as CSV. If you have a compressed archive (.zip, .gz), extract it first. Binary files (images, PDFs) are not supported.

Analysis shows 0 genes

Cause: The first column may contain numeric data instead of gene names, or the file structure doesn't match expectations.

Fix: Ensure the first column contains gene names (text strings). For gene expression matrices, use CT/ZT headers for time-point columns. For generic CSVs, ensure all data columns are numeric.

All eigenvalues near zero

Cause: The data may lack temporal variation — constant or near-constant values across time points.

Fix: Check that your data has meaningful temporal dynamics. Genes with variance < 1e-10 are automatically excluded. If all genes show near-zero eigenvalues, the data may not have sufficient oscillatory or autoregressive signal.

"Reversed hierarchy" in Gearbox Analysis

Cause: Target genes have higher mean eigenvalues than clock genes.

Fix: This is a real finding, not an error. It can occur in disease states, specific tissues, or under experimental perturbations where the normal clock > target hierarchy is disrupted. Investigate which genes are driving the reversal.

Share link doesn't work

Cause: The URL may be incomplete or incorrectly copied.

Fix: Make sure the full URL was copied, including the complete ID string after /shared/. The link format is https://[domain]/shared/[id]. If the link still doesn't work, the analysis may have failed to save — try creating a new share link.

Report download gives 404 or empty file

Cause: The report may not have been generated yet for that particular analysis, or the page doesn't support report downloads.

Fix: Only certain pages have Download Report functionality: Disease Screen, Robustness Suite, and Wearable Analysis. Make sure you've run an analysis on that page before attempting to download. If the download still fails, try refreshing the page and re-running the analysis.

Analysis seems stuck or takes too long

Cause: Very large files (close to the 50 MB limit) or datasets with many thousands of genes may take longer to process.

Fix: Wait for the analysis to complete — results appear all at once when finished. If it appears stuck for more than 2 minutes, try reducing the file size or number of genes. The Genome-Wide Analysis page may take 30–60 seconds for 20,000+ genes.

Edge case diagnostics show many warnings

Cause: The data may have quality issues: trends, non-stationarity, very short time series, or values near the stationarity boundary.

Fix: Warnings are informational — they don't prevent analysis but indicate that results should be interpreted with caution. Common fixes: detrend your data before uploading, ensure at least 8+ time points, and check for batch effects or technical artifacts in the expression measurements.

Page loads but shows no data

Cause: The page may require a dataset selection or analysis to be run first.

Fix: Most analysis pages require you to select a dataset and/or click "Run Analysis" before results appear. Check for a dataset selector dropdown or analysis button at the top of the page. The Dashboard is the best starting point for exploring pre-loaded datasets.
